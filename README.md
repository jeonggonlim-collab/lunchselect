# 오늘 점심 뭐 먹지? 🍽️

팀원들이 모바일로 접속해서 카드를 뽑으며 점심 취향을 투표하면,
응답을 모아 그날의 추천 식당을 보여주는 앱입니다.

## 식당 목록 관리 (restaurants.xlsx)

프로젝트 루트의 `restaurants.xlsx` 파일로 식당 목록을 관리합니다.
열(컬럼) 구성은 다음과 같습니다.

| 식당명 | 종류 | 음식종류 | 거리 |
|---|---|---|---|
| 명동교자 | 한식 | 면류 | 150 |

- **종류**: 한식 / 양식 / 중식 / 일식 / 그외 중 하나로 입력
- **음식종류**: 면류 / 고기 / 피자 / 밥류 / 분식 / 그외 중 하나로 입력
- **거리**: 미터(m) 단위 숫자

투표 카드의 선택지와 다른 값을 입력하면 해당 식당은 추천 점수 계산에서
매칭되지 않으니 위 표기를 그대로 맞춰주세요.

식당 목록을 바꾼 뒤에는 저장만 하면 서버가 파일 수정 시각을 감지해
자동으로 다시 읽어옵니다 (서버 재시작 불필요). 다만 배포 환경(Render)에서는
파일이 깃 저장소 안에 있으므로, **수정 후 커밋 & 푸시해야 배포본에도 반영**됩니다.

## 로컬 실행

```bash
npm install
npm start
```

브라우저에서 http://localhost:3000 접속.

## 외부(모바일)에서 접속되도록 배포하기 — Render

1. 이 폴더를 GitHub 저장소로 올립니다.
   ```bash
   git init
   git add .
   git commit -m "lunch vote app"
   # GitHub에서 새 저장소를 만든 뒤
   git remote add origin <저장소 URL>
   git push -u origin main
   ```
2. [render.com](https://render.com) 에 가입/로그인합니다. (계정 생성은 직접 진행해주세요)
3. 대시보드에서 **New +** → **Web Service** 선택
4. 방금 올린 GitHub 저장소를 연결
5. 설정값
   - Environment: `Node`
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Plan: `Free`
6. **Create Web Service** 클릭 → 몇 분 뒤 `https://xxxx.onrender.com` 형태의 공개 URL 생성
7. 이 URL을 팀원들에게 공유하면 각자 휴대폰 브라우저로 접속 가능

이 저장소에는 `render.yaml`도 포함되어 있어서, Render의 **Blueprint** 기능으로
한 번에 설정을 불러올 수도 있습니다.

### 참고: 무료 플랜 유의사항

- Render 무료 웹 서비스는 일정 시간 요청이 없으면 슬립 상태가 되고,
  다음 요청 시 다시 깨어나는 데 몇십 초 걸릴 수 있습니다.
- 투표 데이터(`data/votes.json`)는 서버 디스크에 저장되며 당일 투표를
  모으는 용도로는 충분하지만, 재배포(코드 푸시) 시 초기화될 수 있습니다.
  여러 날짜의 투표 기록을 영구 보관하려면 별도 데이터베이스 연동이 필요합니다.

## 동작 방식 메모

- 참여자는 이름 입력 후 4개 카테고리(참여/거리/종류/음식종류)의 카드 덱에서
  카드를 하나씩 탭해 선택합니다.
- **참여하겠다**를 고른 사람의 응답만 식당 추천 점수 계산에 반영됩니다.
  (묻어가겠다는 인원 집계에는 포함되지만 메뉴 결정에는 영향을 주지 않습니다.)
- **거리**는 "가깝게"를 고른 인원 수에 비례해 가까운 식당(≤300m, ≤500m)에
  가점을 줍니다. "상관없다"/"새로운 장소"는 현재 별도 필터링 없이
  중립으로 처리됩니다("새로운 장소"를 방문 이력 기반으로 반영하려면
  식당별 방문 기록 데이터가 추가로 필요합니다).
- 같은 기기(브라우저)로 같은 날 다시 투표하면 이전 응답을 덮어씁니다.
